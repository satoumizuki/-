# 卒業制作カウントダウン

卒業日（MVPリリース予定日）から逆算して、卒業制作の各STEP
（README → 画面遷移図 → ER図 → issueベースの開発 → MVPリリース）の
着手目安日を自動で計算し、そのタイミングでそっと背中を押す学習トラッカーです。

バックエンドはなく、単一のHTMLだけで動きます。進捗はブラウザ（localStorage）に保存されます。


# 使い方

1. 初回セットアップ

アプリを開くと「タスク（課題）確認を始めよう！」の画面が出ます。次の3つを入力して「はじめる」を押します。


コース：Web開発スタンダード / Python コースのどちらかを選ぶ
卒業日：MVPリリース予定日を設定する
1週間の学習時間：普段どれくらい勉強するか（初期値20時間。あとで変更できます）


2. 進捗をチェックする

「📍 今ここまで進んだ」ボタンから進捗更新画面に進み、終わったタスク（課題）にチェックを付けます。
チェックした分がホーム画面の集計に反映されます。

3. ホーム画面で状況を確認する


卒業日までの残り日数
週の目標：「〇/〇(曜)までに終わらせたいタスク（課題） 〇個」
コーチからの声かけ：いま着手すべきSTEPを、タイミングに合わせて通知
卒業制作ロードマップ：各STEPの目安時間と着手目安日、チェック状況


4. 学習時間を変える

ホーム画面右上の ⚙ボタン で「1週間の学習時間」を変更できます。
変えると、そのペースに合わせて各STEPの着手目安日が自動で計算し直されます（進捗は消えません）。

5. 最初からやり直す

進捗更新画面の一番下の 「設定をリセット」 で、保存データを消してセットアップからやり直せます
（※進捗もすべて消えます）。


# 通知（コーチ）の仕組み

卒業日と週の学習時間から、各STEPの「着手目安日」を逆算しています。
各STEPにはレビュー・修正の時間として 見積り＋25% の余裕を織り込んでいます。

設計フェーズ（README・画面遷移図・ER図・issue）では、目安日を過ぎても未着手のとき、
声かけのトーンが段階的に上がります。


🔥 目安日が来たら：「そろそろ着手しよう」
👀 目安を4日過ぎても未着手：「もう着手できてる？完璧じゃなくてOK」
🤝 8日過ぎても未着手：「詰まってない？一緒に進めよう」


次のことはしません（急かしすぎない設計です）。


目安日が来ていないSTEPは通知しない（先回りしない）
前のSTEPが終わるまで、次のSTEPは通知しない（1つずつ・順番どおり）
完了したSTEPは通知しない（自動で次へ進む）



# 計算の考え方（あくまで目安です）

このアプリが出す日付や個数は、あらかじめ決めた見積りをもとに機械的に計算した 目安 です。
進み方は人によって違うので、必ず自分の学習計画表と合わせて確認してください。

週の目標（〇個）

残っているタスク（課題）の数を、卒業日までの残り週数で割った「1週間あたりの目安数」です（端数は切り上げ）。
基準は「次の月曜〜日曜」の週で数えています。

STEPの着手目安日

卒業制作の各STEPに、目安の作業時間を設定しています。

STEP目安の作業時間README10時間画面遷移図5時間ER図15時間issueベースの開発10時間MVPリリース140時間

「そのSTEP以降に必要な合計時間 ÷ 1週間の学習時間」で必要な週数を出し、そこに
レビュー・修正ぶんの余裕（＋25%）を足して、卒業日から逆算した日を「着手目安日」としています。
1週間の学習時間（⚙で変更）を変えると、この目安日も自動で変わります。

ここが大事

表示される時間・個数・日付は、あくまで一つの目安です。得意分野や実装の詰まり具合で前後します。
焦らず、自分の学習計画表と照らし合わせて、無理のないペースで進めてください。
卒業日に間に合えばオールOKです🙆


# 注意

進捗データは ブラウザ・端末ごとに別 です（スマホとPCで同期しません）。
複数人で使うときは「各自の端末に保存される」前提を伝えると親切です。
バックエンドはなく、個人情報の送信もありません。
